# Study 3d — Rank-binned NPS Prediction

Tests the hypothesis that calibrating the predicted NPS distribution to
match the empirical multinomial — via rank-preserving binning — gives
NPS-priority a non-degenerate priority structure that the original
continuous formulation does not have under paper calibration.

## Background

Studies 2/3/3b/3c established that the *raw* predicted NPS (Eq. 9) lives
in a very narrow band (std ≈ 0.04, range ≈ 0.5 NPS units), so
`|NPS_hat − 7.5|` is essentially monotonic in throughput → NPS ≡ LRTF.
The realised NPS responses (Eq. 8) span 0–10 with substantial mass on
both sides of 7.5. Study 3d closes this scale mismatch by rank-mapping
the predicted distribution onto the empirical multinomial supplied by
the paper authors (n = 1,898 valid responses, weighted mean 8.83).

The paper itself flagged this exact follow-up (p. 23):

> *"In follow-up studies, the distribution of the individual NPS responses
> should be systematically varied … to identify the conditions under which
> the NPS-based prioritisation discipline will diverge from the LRTF
> discipline."*

## Hypothesis

> **H₃d.** When predicted NPS is rank-binned to match the empirical
> multinomial, NPS-prioritization (a) breaks ≡ LRTF, (b) shifts the
> realised response distribution toward promoters and away from
> detractors compared to LRTF / FCFS / SRTF, and (c) does so at the
> paper's baseline calibration (intercept = 10.22).

## Method — rank-binning

Given a target multinomial `p = (p₀, …, p₁₀)`:

1. Generate cases with raw continuous `predicted_nps` from Eq. 9 (unchanged).
2. Sort cases ascending by `predicted_nps`.
3. Walk sorted list, allocating cases to bins by cumulative rounding:
   bottom `floor(N · p₀)` get bin 0, next `floor(N · p₁)` get bin 1, etc.
4. Assign the result to `case.predicted_nps_binned`.
5. Priority = `|predicted_nps_binned − 7.5|`. Random tie-break inside
   each priority class via a per-case key drawn from `rng_arrivals`.

The transformation is rank-preserving — the case predicted to have the
shortest throughput still gets the highest predicted NPS.

## Experimental design

| Factor | Levels |
|--------|--------|
| **Discipline** | FCFS, LRTF, SRTF, NPS, **NPS_BINNED** |
| **ρ** | 0.0, 0.22, 0.5, 0.85, 1.0 |
| **Sampling mode** | hard |
| **Agents** | 6 |
| **NPS intercept** | 10.22 (paper baseline) |
| **Replications** | 100 |

**Total runs:** 4 × 5 × 100 (LRTF/SRTF/NPS/NPS_BINNED) + 5 × 100 (FCFS, ρ-invariant but kept for paired seeding) = **2,500**.

## Sanity checks

1. Spearman correlation between raw `predicted_nps` and `predicted_nps_binned` must be 1.000 (rank preservation).
2. Per-rep histogram of binned values matches the target multinomial.
3. FCFS results match Study 2 baseline (FCFS does not use the predicted NPS).
4. NPS, LRTF, SRTF results match study3c topic-blind, intercept-10.22 cells.

## Expected outcomes

- NPS_BINNED ≠ LRTF (rank-binning makes the priority structure non-degenerate).
- NPS_BINNED prioritises predicted bin 7/8 first (priority 0.5), which under
  the empirical multinomial is ~14.5% of cases — the predicted passives.
- ~62% of cases are binned to 10 (priority 2.5, large tied block resolved randomly).
- Headline metric: shift in segment shares (detr / pass / prom) versus FCFS / LRTF / SRTF / NPS.
