# Study 4 — Sinusoidal arrival rate sensitivity

Tests whether priority disciplines differentiate more strongly when the
arrival rate oscillates predictably around a fixed long-run mean. The
mean arrival rate is held constant; only the temporal variability changes.

## Background

Studies 3 → 3e found that under the paper's stationary arrival process,
priority disciplines collapse into a tight ±1 pp band around the FCFS
baseline at the optimum. SRTF and NPS_BINNED at f = 0.20 tie within MC
noise on every measured metric at ρ = 1. The diagnosis was that "the
algorithm has little work to do" because queue-length variance is small
in the stationary regime — by day 100 the queue settles into a tight
steady state.

Study 4 tests that diagnosis directly by introducing controlled non-
stationary load: peaks and lulls that grow the queue beyond steady state
during peaks and let it drain during lulls.

## Hypotheses

- **H1 (load-dependent value).** The org-NPS gap between FCFS and the
  best priority discipline grows monotonically in amplitude `A`.
- **H2 (SRTF amplification).** SRTF advantage over LRTF grows with `A`
  because long cases get trapped in deep peak queues, which LRTF
  actively makes worse.
- **H3 (NPS_BINNED vs SRTF).** Non-stationary load is where NPS_BINNED
  could break out of the SRTF tie observed in Study 3e. Direction of
  effect left open.
- **H4 (period sensitivity).** Effects are larger at longer periods
  (28d > 14d) because the queue accumulates more across a peak before
  the lull lets it drain.
- **H5 (structural invariance).** NPS ≡ LRTF still holds at any `A` —
  the monotonicity collapse from Study 3 is independent of arrival-
  process shape.

## Arrival-rate modification

Overlay sinusoidal modulation on the existing year/month/day/weekday
base rate `λ̄(t)`:

```
λ(t) = λ̄(t) · (1 + A · sin(2π · t / P + φ))
```

Long-run mean preserved because ∫sin = 0 over each full period.

Implementation: Lewis thinning with day-bounded `λ_max`, restarting
across day boundaries (where the base rate jumps). At `A = 0` the code
takes a fast path that uses exactly one uniform draw per inter-arrival,
so A=0 cells are seed-equivalent to Study 3e cells of the same (ρ, rep).

## Experimental design

| Factor | Levels |
|---|---|
| **Discipline** | FCFS, LRTF, SRTF, NPS, NPS_BINNED (at f = 0.20) |
| **Amplitude `A`** | 0.00, 0.25, 0.50, 0.75 |
| **Period `P`** | 14 days, 28 days |
| **ρ** | 0.5, 1.0 |
| **Phase `φ`** | 0 (deterministic, simulation starts at rising-edge mean) |
| **Agents** | 6 |
| **NPS intercept** | 10.22 (paper baseline) |
| **SLA** | none |
| **Replications** | 100 |
| **Burn-in** | 30 days (paper-consistent) |
| **Sim length** | 365 days (paper-consistent) |
| **Measurement window** | days 30–365 (335 days, paper-consistent) |

**Total runs:** 5 disciplines × 4 A × 2 P × 2 ρ × 100 reps = **8,000**.

## Sanity checks (printed by `run_experiments.py`)

1. **Mean rate preservation.** `arrivals_after_burnin` constant across A
   at fixed (P, ρ) within MC noise.
2. **NPS ≡ LRTF (H5).** `|orgNPS(NPS) − orgNPS(LRTF)|` ≈ 0 at all cells.
3. **Cyclic-steady-state.** orgNPS in first half vs second half of the
   measurement window — flagged if difference > 0.3 pp at high A.
4. **Baseline arrival-rate variance** diagnostic, printed at startup,
   reports the implicit "effective amplitude" already present in the
   year/month/weekday base rate.

## Run via Docker

From the repo root:

```bash
echo "STUDY=study4" > .env
docker compose up --build
```

On servers with older Docker CLI:

```bash
DOCKER_API_VERSION=1.41 docker compose up --build -d
```

## Run locally

```bash
cd study4

# smoke test (in-process)
python3 simulation.py

# stability pilot at worst-case cell (A=0.75, P=28, FCFS)
python3 run_experiments.py --pilot

# quick smoke (5 reps × 90 days)
python3 run_experiments.py --quick

# full experiment (8,000 runs)
python3 run_experiments.py --workers 8

# generate figures
python3 generate_plots.py
```
